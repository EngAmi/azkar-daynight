# خطة: تذكيرات الأذكار اليومية

نبدأ بأهم ميزة ناقصة: **تذكير يومي بأذكار الصباح والمساء** عبر إشعارات المتصفح — تحويل الموقع من "زيارة عابرة" إلى عادة يومية.

## نطاق المرحلة الأولى (بسيط ويعمل الآن)

- المستخدم يفتح الإعدادات ويفعّل التذكير ويختار وقتين: صباح (مثلًا 7:00 ص) ومساء (مثلًا 5:00 م).
- نطلب إذن الإشعارات من المتصفح.
- عند تفعيلها يُجدوَل الإشعار التالي داخل الجهاز (بدون خادم).
- الإشعار عند النقر يفتح الصفحة مباشرة على جلسة الفترة المناسبة.
- كل الإعدادات محفوظة محليًا (localStorage). لا حساب مطلوب، لا خادم، لا Web Push حاليًا.

## ما سنبنيه

1. **`src/lib/notifications.ts`** — طبقة صغيرة:
   - `requestPermission()`
   - `scheduleNext(times)` تحسب أقرب موعد قادم (صباح أو مساء) وتضبط `setTimeout` — عند إطلاقه تُعرَض إشعار عبر Service Worker (`registration.showNotification`) مع fallback إلى `new Notification` عندما لا يوجد SW (بيئة المعاينة/التطوير).
   - إعادة الجدولة تلقائيًا بعد كل إشعار، وعند تحميل الصفحة، وعند العودة إلى التبويب (`visibilitychange`).

2. **`src/hooks/useReminderSettings.ts`** — قراءة/كتابة الإعدادات في localStorage (`enabled`, `morningTime`, `eveningTime`) مع مزامنة بين التبويبات.

3. **`src/components/ReminderSettings.tsx`** — بطاقة/مودال بسيط داخل الصفحة الرئيسية:
   - مفتاح تشغيل/إيقاف
   - حقلا وقت (`<input type="time">`) للصباح والمساء
   - رسالة توضح أن الإشعارات تعمل فقط بعد فتح الموقع مرة أو تثبيته على الشاشة الرئيسية (قيد المتصفحات بدون Web Push).
   - زر "أرسل إشعار تجريبي" للتأكد.

4. **`src/pages/Index.tsx`** — زر صغير "التذكيرات" بجانب باقي الأزرار يفتح `ReminderSettings`، ونداء `scheduleNext` عند التحميل إن كانت مفعّلة.

5. **معالجة النقر على الإشعار** — نضيف مستمع `notificationclick` بسيط داخل Service Worker المُولَّد؟ vite-plugin-pwa/generateSW لا يسمح بحقن كود مباشرة، لذلك نستخدم `registration.showNotification` مع `data.url`، ونعتمد على السلوك الافتراضي (فتح الأصل). كبديل عملي: نضبط `data: { url: "/?session=morning" }` ونقرأ المعامل في `Index.tsx` لبدء الجلسة تلقائيًا.

## القيود الصريحة للمستخدم

- الإشعارات تعتمد على المتصفح المحلي؛ إذا كان الجهاز مغلقًا أو المتصفح غير نشط لفترة طويلة قد يتأخر الإشعار.
- iOS Safari يتطلب تثبيت الموقع على الشاشة الرئيسية أولًا لدعم الإشعارات (سنعرض تنبيه واضح للمستخدم على iOS).
- الإشعارات الحقيقية تعمل فقط في النسخة المنشورة (Service Worker معطّل في المعاينة).

## المرحلة الثانية (لاحقًا، اختياري)

- Web Push حقيقي عبر VAPID + Edge Function في Lovable Cloud لإرسال الإشعارات حتى بدون فتح الموقع. نناقشه بعد اختبار المرحلة الأولى.
